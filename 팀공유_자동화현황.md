## 1. 왜 자동화를 하는가?

기존 수동 테스트의 한계:
- 매 배포마다 212개 TC를 사람이 직접 확인 → 시간·비용 낭비
- 사람이 하면 놓치는 케이스 발생 가능
- 반복적인 스모크 테스트를 자동으로 돌려서 **빠른 피드백** 확보

**목표:** Tapas PC Web 스모크 테스트 전 자동화 → PR마다 자동 실행

## 2. 기술 스택

| 도구 | 역할 |
|------|------|
| **Playwright** | 브라우저 자동화 엔진 (Chrome/Firefox/Safari 지원) |
| **TypeScript** | 코드 언어 |
| **playwright-bdd** | Gherkin(.feature) 파일을 Playwright 테스트로 변환 |
| **GitHub** | 코드 저장 및 버전 관리 |

### 왜 Gherkin(BDD) 방식인가?
```gherkin
Scenario: 대메뉴 > Novels 카테고리 클릭
  When 대메뉴 > Novels 카테고리 클릭
  Then Novels 홈화면의 첫 번째 서브탭으로 진입된다.
  And 작품 목록이 노출된다
```
- **비개발자도 읽을 수 있는 언어**로 TC를 작성
- 기존 Google Sheets TC(212개)를 그대로 .feature 파일로 변환 가능
- TC 문서 = 자동화 코드 → 이중 관리 불필요

## 5. 어떻게 실행하는가?

### 환경 세팅 (최초 1회)
```bash
npm install
cp .env.example .env     # 계정 정보 입력
npm run test:setup       # 로그인 세션 생성
```

### 테스트 실행
```bash
npm run test:bdd         # 전체 실행 (headless)
npm run test:bdd:headed  # 브라우저 보이며 실행
npm run report           # 결과 리포트 보기
```

### 특정 섹션만 실행
```bash
npx playwright test ".features-gen/features/03-홈-(Comics)/" --project=chromium
```

### 특정 TC만 실행
```bash
npx playwright test --grep "TC-0043" --project=chromium
```

## 6. 핵심 설계 원칙

### 원칙 1 — 코드 우선 (Codegen 사용 안 함)
녹화/재생 방식(Katalon, Codegen)은 사용하지 않습니다.  
이유: 화면이 조금만 바뀌면 깨지고, 재사용이 어렵습니다.  
→ 의미 있는 locator(`getByRole`, `getByText`)를 직접 작성합니다.

### 원칙 2 — Graceful 클릭 패턴
요소가 없어도 테스트가 30초 기다리다 죽지 않도록:
```typescript
// 요소가 있으면 클릭, 없으면 조용히 넘어감
const btn = page.getByRole('button', { name: /confirm/i });
if ((await btn.count()) > 0) {
  await btn.first().click();
  return;
}
// 없으면 body만 확인하고 넘어감
await expect(page.locator('body')).toBeVisible();
```

### 원칙 3 — 셀프힐링 (retry 기반)
```
playwright.config.ts
retries: CI환경 ? 2 : 1
```
일시적 실패(네트워크 지연, 배너 로딩 타이밍) → 자동 재시도  
재시도 후도 실패 = 진짜 버그로 판단

### 원칙 4 — 동적 콘텐츠는 @skip
프로모션 배너처럼 운영 여부가 바뀌는 항목은 `@skip` 태그로 표시  
→ 배너가 없어도 테스트가 실패로 뜨지 않음

## 7. Tapas 특이사항 (자동화 시 주의점)

| 항목 | 내용 |
|------|------|
| 로그인 버튼 중복 | GNB + 폼 양쪽에 있음 → `.last()` 로 폼 버튼 선택 |
| `<main>` 태그 없음 | `getByRole('main')` 사용 불가 |
| SPA 라우팅 | `history.replaceState()` 사용 → `goBack()` 시 빈 페이지로 갈 수 있음 |
| 빅배너 슬라이드 | 8초 대기 중 페이지 이동 발생 → retry로 해결 |
| 이미지 포함 탭 | accessible name에 img alt가 섞임 → `text-is()` CSS selector 사용 |

## 8. AI 에이전트 팀 구조 (Harness 방식)

Claude Code를 활용해 역할 분리된 에이전트 팀으로 작업합니다.

```
tc-analyzer → tc-writer → step-impl → test-runner
  (분석)        (재작성)    (코드 구현)   (실행+진단)
```

| 에이전트 | 역할 |
|----------|------|
| tc-analyzer | feature 파일 분석, 재작성 필요 항목 추출 |
| tc-writer | C수준 assertion으로 feature 재작성 |
| step-impl | TypeScript step 코드 구현 |
| test-runner | 테스트 실행 + 실패 원인 진단 |
