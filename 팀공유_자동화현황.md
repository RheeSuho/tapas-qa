## TC 작성 방식 — Gherkin(BDD)

```gherkin
Scenario: [TPS-042] 대메뉴 > Comics 카테고리 클릭
  When 대메뉴 > Comics 카테고리 클릭
  Then Comics 홈화면의 첫 번째 서브탭으로 진입된다.
  And 작품 목록이 노출된다
```

- **비개발자도 읽을 수 있는 언어**로 TC 작성
- 기존 Google Sheets TC 213개를 `.feature` 파일로 변환
- TC 문서 = 자동화 코드 → 이중 관리 불필요
- 검증 수준: URL 패턴 + 핵심 DOM 요소 존재 확인

## 핵심 설계 원칙

### 1. 코드 우선 (녹화/Codegen 사용 안 함)
Katalon, Playwright Codegen 같은 녹화·재생 방식을 쓰지 않습니다.  
화면이 조금만 바뀌어도 전체가 깨지기 때문입니다.  
대신 의미 있는 locator(`getByRole`, `getByText`)를 직접 작성합니다.

### 2. 요소가 없어도 죽지 않는 클릭 패턴
```typescript
// 요소가 있으면 클릭, 없으면 조용히 통과
const btn = page.getByRole('button', { name: /confirm/i });
if ((await btn.count()) > 0) {
  await btn.first().click();
  return;
}
await expect(page.locator('body')).toBeVisible();
```

### 3. 자동 재시도 (셀프힐링)
```
retries: CI 환경 ? 2회 : 1회
```
네트워크 지연, 배너 로딩 타이밍 같은 일시적 실패 → 자동 재시도  
재시도 후도 실패 = 진짜 버그로 판단

### 4. 동적 콘텐츠는 @skip 태그로 명시
프로모션 배너처럼 운영 여부가 바뀌는 항목은 `@skip` 태그 부여  
실행에서 제외되며, 이유는 주석으로 남겨 추후 해제 기준이 됩니다.

## 로컬 실행 방법

```bash
# 환경 세팅 (최초 1회)
npm install
cp .env.example .env     # 계정 정보 입력
npm run test:setup       # 로그인 세션 생성

# 실행
npm run test:bdd         # 전체 실행 (headless)
npm run test:bdd:headed  # 브라우저 보이며 실행

# 특정 TC만
npx playwright test --grep "TPS-042" --project=chromium

# 리포트 확인
npm run allure           # Allure 대시보드 로컬 열기
npm run report           # Playwright HTML 리포트
```
