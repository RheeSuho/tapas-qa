## 2026-05-13 (15일차) — 신규 TC 추가 및 CI 속도 문제 해결

### 한 일

#### 1. 신규 TC 18개 추가 (TC-0221~TC-0238)
- 02-홈: Completed / Free Access / WUF 서브탭 TC (빅배너, 더보기, 자동슬라이드)
- 05-홈-(Mature): Daily / Popular / New / Completed 서브탭 TC
- 06-홈-(Community): Early Access / Completed 서브탭 TC

#### 2. CI 속도 문제 원인 파악 및 수정
- 원인: `workers: 1` (순차 실행) + `retries: 2` + 신규 TC 실패 → 22분 → 117분 폭증
- 수정: `workers: 2`, `retries: 1` 로 변경
- sharding(2 job 분할) 시도 후 롤백 — 동일 계정 동시 세션 충돌 위험

#### 3. Spotlight 동적 skip 보완
- 기존: Spotlight 탭 링크 없으면 skip
- 문제: 탭은 있지만 콘텐츠(배너)가 없을 때 fail 발생
- 수정: 탭 클릭 후 배너 콘텐츠 없으면 추가 skip 처리
- 적용 대상: Comics / Novels / Mature / Community Spotlight 4개

#### 4. Mature New > Novels 필터 "No results" 처리
- Novels 신작 없을 때 "No results were found." 문구 노출이 기획 의도
- `날짜별 신작 목록이 노출된다` step에 "No results" 문구도 pass로 처리

### 문제 → 해결

| 문제 | 해결 |
|------|------|
| 신규 TC 추가 후 CI 22분 → 117분 폭증 | workers=1→2, retries=2→1 로 수정 |
| Spotlight 탭 있어도 콘텐츠 없으면 fail | 배너 콘텐츠 체크 후 없으면 동적 skip 추가 |
| Mature New Novels "No results" 시 fail | step에 "No results were found." 문구도 정상 처리 추가 |

### 특이사항
- CI는 workers=2, retries=1 단일 job 구조 유지 (sharding은 세션 충돌 위험으로 채택 안 함)
- Spotlight 동적 skip 로직: 탭 없음 → skip / 탭 있고 배너 없음 → skip / 배너 있음 → 검증
- 오늘 regression 결과 분석 중 (run #13)
