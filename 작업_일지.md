## 2026-05-11 (14일차) — 네비게이션 방식 전환 + @skip 해제 + 신규 TC + Slack 개선

### 한 일

#### 1. Spotlight 서브탭 네비게이션 방식 전환
- 기존 `page.goto('menu/X/subtab/1')` URL 번호 직접 진입 방식 → 텍스트 "Spotlight" 링크 클릭 방식으로 교체
- 이유: `subtab/1`이 항상 Spotlight가 아님 (예: menu/3의 Daily는 subtab/16)
- 적용 파일: `steps/홈-카테고리.steps.ts`, `steps/홈.steps.ts`
- Spotlight 링크가 없으면 `test.skip(true, 'OO Spotlight 서브탭 미운영 상태')` 동적 skip 처리

#### 2. 홈 subtab URL 번호 매핑 전면 제거 (홈.steps.ts)
- `HOME_SUBTAB` 상수 및 번호 매핑 완전 삭제
- `홈 > X 서브탭을 클릭한다`: GNB 홈 클릭 후 텍스트 기반 서브탭 탐색으로 전환
- `(Daily|Popular|...) 서브탭 화면이 노출된다`: URL 검증 제거 → 해당 텍스트 요소 visible 검증으로 전환

#### 3. 'OO 홈으로 돌아온다' fallback goto 제거
- `홈-카테고리.steps.ts`의 4개 Then step(`Comics/Novels 홈으로 돌아온다`)
- 기존: `goBack()` 후 실패 시 `goto('/menu/X')` fallback
- 변경: fallback goto 제거, `article a[href*="/series/"]` DOM 검증만으로 assertion

#### 4. 신규 TC 8개 추가 — 기다무/유료 작품 검색 기반 진입
- `features/07-작품홈/진입.feature`: TC-0213~0216 (Comic/Novel × 기다무/유료 작품홈 진입)
- `features/09-뷰어-(Comic)/진입.feature`: TC-0217~0218 (Comic × 기다무/유료 뷰어 진입)
- `features/10-뷰어-(Novel)/진입.feature`: TC-0219~0220 (Novel × 기다무/유료 뷰어 진입)

#### 6. Slack 알림 — 동적 skip 사유 표시 기능 추가
- `scripts/notify-slack.js`에 `dynamicSkips` 수집 로직 추가
- `test.skip(true, '사유')` annotation이 있는 TC를 자동 집계
- Slack 블록에 *Skip 사유* 섹션 신설 — `• [TPS-XXX] TC명 : 사유` 형태로 최대 10건 표시

### 문제 → 해결

| 문제 | 해결 |
|------|------|
| `subtab/1`이 항상 Spotlight가 아님 | URL 번호 방식 폐기 → 텍스트 "Spotlight" 링크 탐색으로 전환 |
| 프로모션 배너 locator — URL 패턴으로 구분 불가 | DOM에서 `div.h-48` (48px 높이 얇은 배너) class로 빅배너와 구분 |
| Activity DOM 파악 불가 (인증 필요) | `node -e`로 Chromium 직접 실행 후 DOM 덤프 → `li.item.js-item`, `a.activity` 확인 |
| 더보기 버튼이 꺽쇠(>) 이미지로 구성됨 | `a[href*="/landing-list/"]` + `img[alt="right arrow"]` 조합으로 정확히 타겟 |
| `{string} 검색 후 작품 클릭` bddgen 오류 | Gherkin step 파라미터 따옴표 처리 수정 |

### 특이사항
URL 번호 의존 방식을 텍스트 기반 클릭으로 전환하는 큰 방향 전환이 있었다. 네비게이션 방식이 바뀌어도 기능 TC의 `ensureOn` 헬퍼는 `goto`를 유지한다는 원칙을 세워서 범위를 명확히 구분했다. 프로모션 배너 같은 동적 콘텐츠는 DOM 속성(`div.h-48`)으로 정확히 감지해서 운영/미운영 여부를 자동 판단하는 패턴을 완성했다.
